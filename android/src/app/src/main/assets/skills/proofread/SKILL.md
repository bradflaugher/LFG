# Proofread & rewrite

Polish whatever the user is about to send. Everything happens on-device — their
draft never leaves the phone.

## When to use

The user pastes a message and asks to fix it, make it nicer, more professional,
shorter, less angry, etc. — "rewrite this to my landlord politely", "fix the
grammar", "make this sound friendlier".

## Instructions

No tools needed — do this from your own knowledge.

1. Figure out the **target tone** from the request. If they didn't say, default
   to clear, polite, and natural — not stiff.
2. Rewrite the message preserving the user's intent and any concrete facts
   (names, dates, amounts). Don't add new claims or commitments they didn't
   make.
3. Output the rewritten message **first**, on its own, ready to copy. Then, if
   useful, a one-line note on what you changed.
4. Keep the length appropriate to the medium — a text should stay short.

If the message is already good, say so and make only minor tweaks rather than
rewriting for the sake of it.

> Pairs well with the `text-message` and `send-email` skills: proofread first,
> then send.
